# ch 2-1 OpenCV 설치와 기초 사용법

## 1. OpenCV를 이용하여 개발된 제품 조사 및 기능 설명하라.

### 1.1 OpenCV란?

OpenCV(Open Source Computer Vision Library)는 1999년 인텔(Intel)이 시작한 오픈소스 컴퓨터 비전 라이브러리이다. 2,500개 이상의 최적화된 알고리즘을 제공하며, C/C++로 작성되었고 Python, Java 등 다양한 언어를 지원한다. 현재 Apache 2.0 라이선스로 배포되며, 대기업은 물론 세계적인 연구기관에서도 광범위하게 사용되고 있다.

### 1.2 OpenCV 기반 주요 제품 및 활용 사례

**① Tesla Autopilot / 자율주행 자동차**

Tesla를 비롯한 자율주행 차량 개발사에서 OpenCV를 활용한다. 차선 검출(Lane Detection), 교통 표지판 인식, 보행자 감지, 주변 차량 추적 등의 기능에 OpenCV의 이미지 처리 및 객체 검출 알고리즘이 핵심적으로 사용된다.

**② Facebook / Meta의 얼굴 자동 태깅**

Facebook은 사용자가 사진을 업로드하면 자동으로 얼굴을 검출하고 태그를 제안하는 기능에 OpenCV 기반 얼굴 인식 기술을 활용한다. `cv2.CascadeClassifier` 등의 알고리즘으로 얼굴 위치를 검출하고, 딥러닝 모델과 결합하여 개인을 식별한다.

**③ Google Maps / 위성 지도 이미지 스티칭**

Google Maps의 위성 이미지 및 스트리트 뷰에서 여러 카메라 영상을 하나로 합치는 이미지 스티칭(Image Stitching) 기술에 OpenCV가 사용된다. `cv2.Stitcher` 클래스를 통해 파노라마 이미지를 생성할 수 있다.

## 2. 소프트웨어 분야의 용어인 라이브러리의 의미를 자세히 설명하라.

### 2.1 정의

라이브러리(Library)란 특정 기능을 수행하는 함수, 클래스, 상수, 데이터 타입 등의 코드 모음을 미리 작성하여 재사용 가능한 형태로 묶어놓은 파일이다. 개발자는 라이브러리를 자신의 프로그램에 포함시켜 해당 기능을 직접 구현하지 않고도 사용할 수 있다.

### 2.2 핵심 특징

- **재사용성(Reusability)**: 한 번 작성된 코드를 여러 프로젝트에서 반복 사용할 수 있다.
- **모듈성(Modularity)**: 프로그램의 기능을 독립적인 모듈로 분리함으로써 코드의 유지보수와 관리가 용이해진다.
- **추상화(Abstraction)**: 라이브러리 사용자는 내부 구현 세부사항을 알 필요 없이 공개된 인터페이스(API)만으로 기능을 활용할 수 있다.

### 2.3 라이브러리의 종류

| 종류 | 설명 | 예시 |
| --- | --- | --- |
| 표준 라이브러리 | 프로그래밍 언어에 기본 포함된 라이브러리 | C의 `stdio.h`, Python의 `os`, `math` |
| 서드파티 라이브러리 | 외부 개발자/조직이 제공하는 라이브러리 | OpenCV, NumPy, Boost |
| 정적 라이브러리 | 컴파일 시 실행파일에 포함되는 라이브러리 | `.lib` (Windows), `.a` (Linux) |
| 동적 라이브러리 | 실행 시 메모리에 로드되는 라이브러리 | `.dll` (Windows), `.so` (Linux) |

## 3. 오픈소스(Open Source)의 의미를 설명하라.

### 3.1 정의

오픈소스(Open Source)란 소프트웨어의 소스코드를 일반에 공개하여 누구나 자유롭게 열람, 수정, 배포할 수 있도록 허용하는 소프트웨어 개발 및 배포 방식이다. 오픈소스 이니셔티브(OSI)가 정한 정의에 따르면, 오픈소스 소프트웨어는 단순히 소스코드 공개를 넘어 자유로운 재배포, 파생 작업물 생성, 차별 없는 사용 등의 조건을 충족해야 한다.

### 3.2 오픈소스의 핵심 원칙

- **소스코드 공개**: 누구나 소스코드를 읽고 학습
- **자유로운 수정**: 사용자의 필요에 맞게 코드를 변경
- **자유로운 재배포**: 원본 또는 수정된 버전을 타인에게 배포
- **라이선스 준수**: MIT, Apache 2.0, GPL, BSD 등의 오픈소스 라이선스 조건 준수

### 3.3 주요 오픈소스 라이선스 비교

| 라이선스 | 상업적 이용 | 수정 코드 공개 의무 |
| --- | --- | --- |
| MIT | 허용 | 없음 |
| Apache 2.0 | 허용 | 없음 |
| GPL v3 | 허용 | 있음 (파생물도 GPL) |
| BSD | 허용 | 없음 |

### 3.4 오픈소스의 장점

- **비용 절감**: 라이선스 비용 없이 소프트웨어를 사용
- **투명성**: 소스코드를 직접 검토하여 보안 취약점이나 버그를 확인 가능
- **커뮤니티 기반 발전**: 전 세계 개발자가 기여하여 빠른 발전과 품질 향상
- **교육적 가치**: 실제 프로덕션 수준의 코드를 학습 자료로 활용
- **벤더 종속 탈피**: 특정 회사의 제품에 종속되지 않고 자유롭게 기술을 선택

## 4. Windows 운영체제의 환경변수 중 Path 변수의 기능을 설명하라.

### 4.1 환경변수란?

환경변수(Environment Variable)란 운영체제가 프로세스를 실행할 때 참조하는 시스템 전역 설정값이다. 변수 이름과 값의 쌍으로 저장되며, 실행 중인 프로그램이 운영체제의 설정 정보를 조회할 수 있도록 한다.

### 4.2 Path 변수의 정의

Path 변수는 Windows 운영체제에서 가장 중요한 환경변수 중 하나로, 실행 파일(.exe, .bat, .cmd 등)을 검색할 디렉터리 경로 목록을 저장한다. 각 경로는 세미콜론(`;`)으로 구분된다.

### 4.3 Path 변수의 동작 원리

사용자가 명령 프롬프트(CMD)나 PowerShell에서 프로그램 이름을 입력하면, 운영체제는 다음 순서로 실행 파일을 찾는다.

1. 현재 작업 디렉터리에서 해당 파일을 검색한다.
2. 파일이 없으면, Path 변수에 등록된 디렉터리를 왼쪽부터 순서대로 검색한다.
3. 일치하는 실행 파일을 찾으면 해당 파일을 실행한다.
4. 모든 경로를 검색해도 찾지 못하면 "'xxx'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램, 또는 배치 파일이 아닙니다."라는 오류 메시지를 출력한다.

### 4.4 Path 변수의 종류

시스템 변수 : 모든 사용자에게 적용
사용자 변수 : 해당 사용자에게만 적용

## 5. 프로그램 개발 과정인 편집, 컴파일, 링크, 실행 단계를 각각 설명하라. 특히, 링크단계에서 라이브러리의 역할에 대하여 설명하라.

### 5.1 편집(Editing)

편집 단계는 프로그래머가 텍스트 에디터나 IDE(통합 개발 환경)를 사용하여 소스코드를 작성하는 과정이다.

- **입력**: 프로그래머의 아이디어와 알고리즘 설계
- **출력**: 소스 파일

### 5.2 컴파일(Compiling)

컴파일 단계는 컴파일러(Compiler)가 사람이 읽을 수 있는 소스코드를 기계가 이해할 수 있는 오브젝트 코드(Object Code)로 변환하는 과정이다.

- **입력**: 소스 파일
- **출력**: 오브젝트 파일
- **세부 과정**: 전처리(Preprocessing**)**  → 구문 분석(Parsing) → 코드 생성(Code Generation)  → 최적화(Optimization)
- **컴파일 에러**: 문법 오류, 타입 불일치, 미선언 변수 등이 이 단계에서 검출

### 5.3 링크(Linking)

링크 단계는 링커(Linker)가 하나 이상의 오브젝트 파일과 라이브러리를 결합하여 하나의 완전한 실행 파일을 생성하는 과정이다.

- **입력**: 오브젝트 파일들 + 라이브러리 파일들
- **출력**: 실행 파일

### 5.4 링크 단계에서 라이브러리의 역할

**① 표준 함수 제공**

프로그래머가 `printf()`, `scanf()`, `malloc()` 같은 표준 함수를 호출하면, 컴파일러는 해당 함수의 "선언"만 확인하고 오브젝트 파일을 생성한다. 함수의 실제 구현 코드는 표준 라이브러리(예: `libc`)에 들어 있으며, 링커가 이를 찾아 오브젝트 파일과 결합한다.

**② 외부 라이브러리 연결**

OpenCV의 `cv::imread()`, `cv::imshow()` 같은 함수를 사용할 경우, 해당 함수의 구현은 OpenCV 라이브러리 파일(예: `opencv_world4100.lib`)에 있다. 링커는 오브젝트 파일에서 호출된 OpenCV 함수의 심볼을 해당 라이브러리에서 찾아 연결한다.

**③ 중복 코드 제거**

여러 소스 파일이 같은 라이브러리 함수를 사용하더라도 라이브러리의 코드는 한 번만 포함되므로 실행 파일의 크기를 줄이고 메모리를 절약할 수 있다.

**④ 미해결 심볼 오류 방지**

링커가 호출된 함수의 구현을 라이브러리에서 찾지 못하면 "unresolved external symbol" 오류가 발생한다. 이는 라이브러리 경로가 설정되지 않았거나 필요한 라이브러리가 누락된 경우에 발생한다.

### 5.5 실행(Execution)

실행 단계는 운영체제의 로더(Loader)가 실행 파일을 메모리에 적재하고 프로그램을 시작하는 과정이다.

- **입력**: 실행 파일
- **과정**:
    1. 운영체제가 실행 파일을 메모리에 로드한다.
    2. 동적 라이브러리(DLL)가 필요한 경우 함께 로드한다.
    3. 프로그램의 진입점(main 함수)부터 코드를 실행한다.
- **런타임 에러**: 메모리 부족, 0으로 나누기, 파일 미발견 등 실행 중에 발생하는 오류이다.

## 6. 정적 라이브러리(SLL)와 동적 라이브러리(DLL)

### 6.1 정적 라이브러리 (Static Library, SLL)

**정의**: 정적 라이브러리는 컴파일된 오브젝트 코드의 모음으로, 링크 시 실행 파일 안에 직접 복사되어 포함되는 라이브러리이다.

**파일 확장자**: Windows: `.lib,` Linux/macOS: `.a` 

**링크 시점: 빌드 타임(Build Time) — 정적 링크**

### 6.2 동적 라이브러리 (Dynamic Link Library, DLL)

**정의**: 동적 라이브러리는 실행 파일과 분리된 독립적인 파일로 존재하며, 프로그램 실행 시점에 메모리에 로드되어 연결되는 라이브러리이다.

**파일 확장자**: Windows: `.dll` (Dynamic Link Library), Linux: `.so` (Shared Object)

**링크 시점:** 프로그램이 시작될 때 운영체제의 **로더**가 자동으로 DLL을 로드, **런타임** 중 필요한 시점에 코드에서 명시적으로 DLL을 로드

### 6.3 정적 라이브러리와 동적 라이브러리 비교 요약

| 비교 항목 | 정적 라이브러리 (SLL) | 동적 라이브러리 (DLL) |
| --- | --- | --- |
| **확장자** | `.lib` / `.a` | `.dll` / `.so`  |
| **링크 시점** | 빌드 타임 (컴파일+링크 시) | 로드 타임 또는 런타임 (실행 시) |
| **실행 파일 크기** | 큼 (라이브러리 코드 포함) | 작음 (참조 정보만 포함) |
| **실행 시 의존성** | 없음 (독립 실행 가능) | DLL 파일 필요 |
| **업데이트** | 전체 재빌드 필요 | DLL만 교체 가능 |
| **메모리 사용** | 프로그램마다 중복 로드 | 여러 프로그램이 공유 |
| **배포 편의성** | 단일 파일 배포 가능 | DLL 동봉 필요 |
| **대표 사용 사례** | 임베디드 시스템, 독립 실행형 도구 | 대규모 응용 프로그램, 플러그인 시스템 |
